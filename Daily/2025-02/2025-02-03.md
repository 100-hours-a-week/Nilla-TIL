### 새로 배운 내용
#### 네트워크 강의 한줄 정리
|  | 정의 | 비고 |
| --- | --- | --- |
| 네트워크 | 장치를 이어 데이터를 주고 받을 수 있게 한 통신망 | - 인터넷은 전세계의 작은 네트워크들이 모여 하나의 큰 네트워크를 구성한 것 |
| 랜(LAN) | 비교적 가까이 있는 거리 내에서 여러 시스템을 연결(스위치에 연결되어있는)하는 네트워크 | - 내부 로컬 망, 스위치 허브 사용, 비용 발생 <br>- star형, 링형, 버스형 등으로 구성됨. 일반적으로 버스형을 많이 사용 |
| 왠(WAN) | Wide Area Network = 2개 이상의 LAN을 연결한 네트워크
<br>지리적으로 떨어진 네트워크를 연결(광범위) | - 구성된 LAN은 또 다른 LAN과 연결하기 위하여 장거리 전송망이 필요.  <br>- Router, 광케이블, 위성 등을 이용하여 WAN 간 연결 |
| 허브 | 네트워크 장비와 장비를 연결하여 데이터를 전송해주는 장치 | - 물리계층(1계층) 장비<br>- 브로드캐스트 방식 사용<br>- 충돌 가능성이 높음 |
| 대역폭 | 1초당 처리할 수 있는 데이터의 양 | - 대역폭이 클수록 전송이 잘된다. <br>- bps(bit per second) 1초에 몇 “비트”를 전송하는지에 대한 단위라 <br>`bps = (용량 X 8) / 처리시간` 으로 계산한다. |
| 스위치  | 네트워크 장비와 장비를 연결하여 데이터를 전송해주는 장치 | - 허브와 비슷하게 네트워크 회선을 연결시켜주나 데이터를 보낼 장치를 고를 수 있음<br>- MAC 테이블을 관리할 수 있음<br>- 대역폭을 확대하는 역할도 함 |
| 비트 | 데이터의 최소 단위 | - 0과 1로 구성되어 있으며 8비트 = 1바이트 |
| 프로토콜 | 네트워크 장비들이 통신하기 위한 표준을 정한 것 | - 제조사마다 다른 프로토콜을 사용하던 시절이 있었으나 그 때문에 다른 제조사끼리 통신에 문제가 생김.<br>- ISO에서 OSI 7계층이라는 표준을 정함<br>- TCP, IP, HTTP, FTP, SMTP 등 다양한 프로토콜이 존재 |
| OSI 7계층 | 네트워크 통신 과정을 7단계로 나눈 표준 | - ISO에서 정한 표준<br>- 통신의 과정을 단계별로 파악하기 위해<br>- 응용, 표현, 세션, 전송, 네트워크, 데이터링크, 물리<br><br> - 7계층: 응용 계층, 사용자가 네트워크를 이용하는 프로그램<br>- 6계층: 표현 계층, 데이터 형식 변환, 암호화/복호화<br>- 5계층: 세션 계층, 연결 설정 및 인증<br>- 4계층: 전송 계층, 신뢰성 있는 정확한 데이터 전달<br>- 3계층: 네트워크 계층, 네트워크 장치 간의 경로 선택과 데이터 전송<br>- 2계층: 데이터 링크 계층, 물리적인 연결을 통해 오류 없는 데이터 전달<br>- 1계층: 물리 계층, 전기 신호를 이용해서 케이블로 데이터 전 |
| TCP/IP 4계층 | OSI 7계층 보다 단순화된 네트워크 표준 모델 | - 응용 계층, 전송 계층, 인터넷 계층, 네트워크 인터페이스 계층<br><br>- 응용 계층(7, 6, 5): 사용자 프로그램이 직접 사용하는 계층<br>- 전송 계층(4): 신뢰성 있는 데이터 전달<br>- 인터넷 계층(3): 네트워크 장치 간의 경로 선택과 데이터 전송<br>- 네트워크 인터페이스 계층(2, 1): 실제 네트워크를 통해 데이터를 전송 |
| 캡슐화 | OSI 7계층에서 포트, IP주소, MAC주소 등의 정보를 담은 헤더를 데이터에 붙여나가는 과정 | - 송수신자의 통신을 위한 정보(header)를 담아 전송하기 위해 사용한다 |
| 역캡슐화 | 데이터 수신시 일어나는 일로 붙어있는 각 계층의 헤더를 제거하며 윗 계층으로 이동하는 과정 |  |
| 트레일러 | 데이터의 끝을 알리는 추가적인 정보 | 오류 검출 및 데이터 무결성을 위해 사용됨 |
| VPN | Virtual Private Network = 인터넷을 통해 사용자와 원격 서버 간에 보안 터널을 만드는 것 |  |
| 이더넷 | 네트워크(랜)을 유선으로 연결하는 방식 | - 물리, 데이터 링크 계층 |
| CSMA/CD | 이더넷에서 장치가 서로 접속하는 방식을 관리하는 방법 | - 데이터를 전송할 때 충돌을 감지하고 해결하는 방법 <br>- 동작 방식 <br>1. 네트워크가 사용 중인지 확인<br>2. 네트워크가 사용 중이지 않으면 데이터 전송 시작<br>3. 다른 장치도 전송할 시, 충돌 발생<br>4. 충돌 발생 시, backoff time을 무작위로 설정<br>5. backoff time이 지나고 다시 전송 시작 |
| MAC | LAN카드에 할당된 주소 | 전세계에 하나밖에 존재하지 않는 고유한 값<br>48비트로 이루어져있다.<br>사실 고유한 값은 아닐 수 있다. (소프트웨어 적으로 수정할 수 있기때문에) → 이걸로 ARP스푸핑도 가능<br>ex) FF:FF:FF:FF:FF:FF |
| ARP | IP 주소와 MAC 주소를 매핑하는 프로토콜 | 동일 네트워크 내에서만 사용되며, ARP 테이블을 관리 |
| IP | 인터넷에 연결되어 있는 모든 장치를 식별할 수 있도록 각각의 장치에 부여된 고유 주소 |  |
| 브로트캐스트 | 로컬 랜 상에 모든 네트워크에 정보를 보내는 방식 | - 전체에 전송하기 때문에 전체 트래픽 증가 <br>- 성능이 떨어질 수 있다 |
| 유니캐스트 | 하나의 송신자가 하나의 특정 수신자에게 데이터를 전송하는 일대일 통신 방식 | - 맥 주소 기반 |
| 멀티캐스트 | 특정 그룹에만 데이터를 전송하는 방식 |  |
| 파싱(parsing) | 데이터를 분석하고 원하는 형태로 변환하는 것 | - 역캡슐화는 파싱의 일종 |
| METHOD | HTTP 요청 방식으로 클라이언트가 서버에 요청을 보낼 때 사용하는 방식 | GET(데이터 조회), POST(데이터 전송), PUT(데이터 수정), DELETE(데이터 삭제) 등 |
| Keepalive | TCP 연결을 유지하여 단일 TCP 통신으로 여러번 통신하는 방법 | - HTTP의 connectionless 특성 때문<br>- 매번 TCP를 연결하고 끊지 않아 트래픽을 감소 |
| DNS | ip주소를 도메인 네임으로 변환함으로써 보다 편리한 접근을 가능하게 하는 시스템 |  |
| DHCP | ip를 자동 할당해주는 프로토콜 |  |
| HTTP | HTML, 이미지, 구조화 된 문서 등을 전송시키기 위한 웹 프로토콜 |  |
| 헤더 | 데이터 패킷의 메타정보를 포함하는 부분 | 송신자, 수신자 정보, 프로토콜 유형 등을 포함 |
| 라우팅 | 네트워크에서 데이터를 목적지까지 가장 효율적으로 전달하기 위한 경로를 설정하는 프로세스 |  |

#### 헷갈리는 개념 학습
- CDN(Content Delivery Network) : 분산된 서버들을 연결한 네트워크. 지리적 제약 없이 전 세계 사용자에게 빠르고 안전하게 콘텐츠를 전송하는 기술로, 캐시 서버가 콘텐츠를 전달한다. 

- 리피터, 스위치, 허브?
  - 리피터는 **1단계 물리 계층에** 위치하며, 신호를 멀리 보내기 위한 증폭 장치다. 요즘은 다른 네트워크 장비에 탑재되어 있어 잘 사용하지 않는다.
  - 허브는 리피터 역할을 하며, **1단계 물리 계층에** 위치한다. 리피터와 다르게 장비를 연결하여 데이터를 모든 연결된 장치에게 보낼 수 있다. 
  - 스위치는 **2단계 데이터 링크 계층**에 위치하며, 네트워크 세그먼트를 연결해주는 브리지와 비슷하다. 또한 허브와 다르게 독립된 충돌 도메인을 제공하므로, 네트워크의 효율성이 향상된다. 링크 어그리게이션을 통해 최대 대역폭을 확장하기도 한다.
