| http | Hypertext(문자를 뛰어넘는 기능) Transfer(전송) Protocol(규약) = HTML 파일, 이미지, 데이터 등을 전송하기 위한 **표준화된 통신 규약** | - 웹 서버와 클라이언트가 Hypertext를 쉽게 주고 받기 위하여 사용한다. <br>- **stateless**: 기존 상태를 저장하지 않는 특성. 수많은 요청을 받기 때문에 모든 요청에 대한 정보를 저장하면 서버에 부하가 온다.  <br>- **connectionless**: 한 번의 요청에 한 번의 응답을 해준 후, 응답을 끊는 특성 |
| http message | HTML, 이미지와 같은 파일과 구조화된 텍스트를 전송하기 위해 서버와 클라이언트가 통신할때 사용하는 메세지 | 헤더(스타트라인, HTTP헤더, 빈 줄 / entity header): 메세지의 기본 정보

| IP | 인터넷에서 사용하는 주소 | - **장치를 식별해 통신**할 수 있게 한다.<br>- 기존에는 IPv4를 사용하였지만, IP주소 부족으로 인해  공인IP,사설 IP로 나누어서 사용하게 되었다. IPv6도 쓰이고 있다.<br>- 고정여부에 따라 유동IP, 공인 IP도 있다.<br>- 데이터를 목적지까지 전달하는 역할을 하며, **신뢰성을 보장하지 않음** |
| DNS | Domain Name System = 도메인을 IP 주소로 변환하는 시스템 | - IP를 그대로 사용하는 경우 너무 복잡하기 때문에 사용자는 도메인([google.com](http://google.com/) 등)을 사용하게 된다. 이를 ip와 매핑시켜 주는 시스템 |
| Proxy Server | 클라이언트&서버 - 인터넷 사이의 중개 서버 | - 서버의 부하를 막거나 은닉하고, 클라이언트의 경우 특정 ip접속을 차단하고, 로드밸런싱 등의 목적으로 사용된다.  |
| WS(Web Server) | HTTP 요청에 응답을 해주는 프로그램 | - 주로 정적페이지를 응답해주는 서버 <br>- 성능 최적화, 트래픽 관리<br>- CGI(Common Gateway Interface)는 웹 서버 - 외부 스크립트 사이의 인터페이스 표준이 되어준다.  |
| WAS | 동적인 웹 페이지와 어플리케이션 서비스를 제공 | - WS와 DB 사이에 있음 <br>- 동적 정보(사용자정보) 제공  |
| 3-way handshake | TCP 통신에서 신뢰성 있는 연결을 설정하는 절차 | 1. SYN: 클라이언트가 서버에 연결 요청. 이 때 SEQ(Sequence Number)가 포함 <br>2. SYN-ACK: 서버가 클라이언트의 요청을 승인, SYN-ACK 패킷을 클라이언트에게 전송.  이 때 SEQ(클라이언트)+1을 ACK 번호로 설정.<br>3. ACK: 클라이언트가 서버의 응답을 받고 ACK 패킷을 전송. 이 때 클라이언트는 SEQ(서버)+1을 ACK 번호로 설정하여 응답. |
| HTTP Cache | 일부 리소스를 웹 브라우저 또는 가까운 서버에 저장해두고 불러와서 쓰는 방식 | -서버의 부하를 줄이고 효율을 높이기 위해 사용한다. <br>-개발 및 유지보수 효율성도 높아짐.  |
| SSR | 서버에서 모든 데이터를 작성하여 클라이언트로 전송하는 방법 | - 서버로부터 데이터를 받아와 실시간으로 동적인 html을 클라이언트에게 전송한다 <br>- 대표적인 SSR로는 Next.js가 있다. |
| CSR | 브라우저에서 렌더링 하는 것 | - 검색엔진 최적화가 어렵다 <br>- 초기 로딩 속도가 느리다 <br>- 대표적인 CSR로는 React.js가 있다. |
| SSG | 웹사이트의 페이지를 빌드 타임에 미리 생성하여 정적인 HTML 파일로 만들어 배포하는 방식 | - 변경 사항이 적은 페이지를 SSG로 사용한다. <br>- 페이지 로딩 속도가 빠르고, 서버 부하가 적다. <br>- 대표적인 SSG로는 Gatsby, Next.js가 있다. |
| Storage | 웹 서버나 로컬 서버에 저장해놓는 공간 | - 쿠키와 다르게 영구적/세션 종료시까지의 만료시간을 갖고, 지정된 domain내에서 유효하다. <br>- 브라우저에서만 접근할 수있고, 스크립트로 접근이 가능하다. |
| HTTP 자격증명 헤더 | HTTP 요청에서 사용자 인증 정보를 전달하는 헤더| - 기본 인증, 다이제스트, Bearer토큰, OAuth <br>- 사용자의 인증을 확인하고 인가하기 위해 사용 <br>- http Authorization 헤더에 인증 정보를 인코딩하여 정보를 담아 서버에 요청한다 |
| OAuth | 타사 서비스를 통해 인증 할 수 있는 개방형 표준 프로토콜 | 카카오, 구글 로그인 등 소셜 로그인에 사용된다.|
| Server Cache | 웹 서버가 데이터를 빠르게 제공하기 위해 자주 사용되는 정보를 임시로 저장하는 장소 | - 로컬 캐시: 각 서버 인스턴스에 독립적으로 존재, 서버의 리소스를 사용하여 데이터를 캐싱 <br>- 글로벌 캐시: 여러 서버 인스턴스 간에 공유되며, 네트워크를 통해 접근 가능한 별도의 캐시 서버에서 데이터를 관리 |
| Apache | 오픈 소스 웹 서버 프로그램  | - 리소스 사용이 많으나 복잡한 동적 컨텐츠 처리에 강점이 있고, 적은 연결 수에서 안정적이다.<br>- 스레드 기반이다. <br>- 다양한 동적 모듈을 쉽게 추가 할 수 있다.  |
| Ngnix | 비동기 이벤트 기반 구조의 웹 서버 아키텍처 | - 비동기 처리 기반으로 여러 이벤트의 높은 동시성 처리<br>- 적은 메모리 사용<br>- 정적 파일을 제공<br>- 이벤트 기반 |
| wireshark | 네트워크 패킷을 캡처하고 분석하는 패킷을 분석하는 프로그램 | - 전송구간을 확인하는데 사용한다.<br>- 네트워크 문제를 해결할 때 사용한다. |
| CORS | 교차 출처 자원 공유 | - 웹 페이지 상의 제한된 리소스를 최초 자원이 서비스된 도메인 밖의 다른 도메인으로부터 요청할 수 있게 허용하는 구조 <br>- 기본적으로 브라우저는 보안의 이유로 다른 출처(Origin)에서의 API 요청을 차단한다. 이를 허용하는 구조 |
