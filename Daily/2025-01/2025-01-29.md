|  | 정의 | 비고 |
| --- | --- | --- |
| 클라이언트 | 다른 컴퓨터 시스템상의 원격 서비스에게 서비스나 자원, 정보를 요청하는 응용프로그램이나 서비스 | 서버로부터 데이터나 서비스를 요청하고 응답 받기 위해 사용 |

| pm2 | Node js 기반 서버의 프로세스를 관리해주는 도구 | 프로세스를 관리하고, 중단 없는 서버 운영을 가능하게 하기 위해서. <br>애플리케이션의 안정성과 성능을 향상하고, 복잡한 운영 환경도 쉽게 관리 가능하다. <br> npm install pm2 -gl<br>pm2 start app.js |
| 코드 컨벤션 | 코드를 작성할 때 사용하는 규약 | 여러 개발자가 협업할 때 체계적으로 코드를 짤 수 있게 하기 위해서 사용한다 <br>에슬린트 - 코드의 오류를 찾아냄 <br> 프리티어 -코드를 깔끔하게 정리해 이해하기 쉽게 만든다.  |
| HTTP | HTML, 이미지와 같은 파일과 구조화된 텍스트를 전송할 때 사용하는 통신 프로토콜 | 웹서버와 클라이언트간의 표준화된 규칙을 사용하여, 매번 새로운 통신 방법으로 하지 않고 정형화되도록 한다. 웹개발시 기본으로 사용할 수 있다 <br>**Stateless** :  이전 상태를 보존하지 않는 방식. 클라이언트의 상태 정보를 저장하면 서버에 부하가 되고, 여러개의 서버가 요청을 처리할 수도 있기 때문에 사용한다.<br>**Connectionless** : 비연결성. 클라이언트가 서버에 요청을 한 후 응답을 받으면 바로 연결을 끊는다. 각 요청을 독립적으로 처리하여 동시에 많은 사용자의 요청을 처리할 수 있고, 동시 연결을 유지할 필요가 없어 자원을 효율적으로 사용할 수 있다.  |
| http message | HTML, 이미지와 같은 파일과 구조화된 텍스트를 전송할 때 사용하는 클라이언트와 서버가 사용하는 통신 메세지 | 1. 스타트라인 <br>2. HTTP헤더 <br>3. 빈줄 <br> 4. 바디  <br>로 구성된다. <br>클라이언트-서버간 데이터를 교환하는 방법을 간단하고 확장성있게 표준화하기 위해서 사용한다.  |
| http Request method | HTTP를 사용할 때 맨 윗줄에 작성되는 요청하는 목적을 나타내는 메시지를 정하는 규약 | 빠르게 요청을 파악하기 위해서 사용된다.<br>GET(데이터 받아오기), POST(데이터 보내기), DELETE(데이터 삭제), PUT(데이터 수정), PATCH(데이터 일부 수정)이 존재한다.  |
| http status code | (웹통신 규약에서) 서버가 http 응답을 보낼 때 그 응답 결과(요청에 대한 처리 결과)의 종류에 대한 분류 코드 | 200번대 : 정상적으로 처리되었음 <br>300번대 : 리디렉션이 필요함 <br>400번대 : 클라이언트의 요청이 잘못되었음 <br>500번대 : 서버가 클라이언트의 요청을 잘못 처리함 |
| URL | 리소스의 위치와 종류를 나타내는 고유한 주소 | 원하는 정보에 정확하고 빠르게 접근하기 위해서 사용된다. |
| Query String | 클라이언트가 서버에게 추가 정보를 보내기 위해 사용하는 문자열 | 서버에게 검색, 필터링, 인피니티 스크롤링 등 기능 구현을 위해 추가 정보를 보낸다. <br>url의 뒤에 붙여 요청을 보낼때 같이 보내는 파라미터 <br>? key = value & key = value & … 형태로 사용한다.  |
| Path variable | url의 pass, 경로 부분에 포함되어 특정 리소스를 식별하는 만드는 변수 | 특정한 리소스에 접근할 때 사용한다.  <br>{} 안에 넣어서 사용한다. post/{id}와 같은 형식으로 사용하여 게시글마다 id를 부여해주는 방식을 예시로 들 수 있다.  |
| SQL | (데이터베이스 시스템에서 자료를 처리하는데 사용되는) 구조화된 데이터 질의문  | 데이터베이스에게 데이터의 조회, 추가 등을 요청하기 위해 사용된다. DDL(데이터 정의어), DML(데이터 조작어), DCL(데이터 제어어)로 나뉘어진다.  |
| 디자인 패턴 | 코드에서 반복되는 문제를 해결하기 위해 미리 만들어진 구조적인 해놓은 코드 설계 방식(해결책) | 구조적인 문제를 해결하고, 코드의 재사용성을 높이기 위해 사용한다.  <br>예시로 루트 컨트롤러(아키텍처 패턴)가 존재한다. 루트 - 컨트롤러 - 서비스 - 모델이 이어져있는 로직이다.  |
| 데이터베이스 | 자료를 정보로써 활용하기 위해 구조화된 데이터들의 모음 | 1. 정보를 효율적으로 관리하고 사용, 공유하기 위해 사용한다.  <br>2. 데이터를 반영구적으로 저장할 수 있다. |
| RDB(Relational Database) | 자료를 정보로써 활용하기 위해 테이블이라는 형태 관계를 맺은 데이터들의 모음 | 구조화하고 체계적으로 관리하기 위해 사용한다.  <br>행과 열을 이용해 데이터를 표시하므로 관리하기에 용이하다. |
| 정규화 | 데이터베이스에서 데이터의 중복을 줄이고 체계화(구조화)하는 프로세스 | 데이터의 일관성과 정확성을 보장하기 위해 사용한다. <br>1NF : 데이터는 각 필드(열)마다 하나의 값만 있어야 한다.  <br>2NF : 주 키의 일부만을 사용하여 다른 속성을 결정할 수 없어야 한다. <br>3NF : 모든 열은 기본 키(각 행을 고유하게 식별하는열)에만 직접적으로 연결되어 있어야한다.  |
| bcrypt | blowfish 알고리즘을 기반으로 한 단방향 암호화 해시함수 | 비밀번호의 보안을 강화하고 안전히 저장하기 위해서 사용한다. <br>솔팅을 사용해 비밀번호에 무작위 데이터를 추가하고 암호화할 수있다.  |
| 인증  | 서버에 접속해도 안전한 사용자인지 확인하는 것 → 사용자의 신원을 확인하는 과정 | 사용자의 신원을 확인해 서비스의 보안을 강화한다. <br> 1. http / 2. 세션 / 3. **jwt(json web token)**을 이용할 . 수 있다. |
| 인가 | 특정 권한을 가지고 접근할 수 있는 데이터인지 결정하는 것 → 인증된 사용자에게 접근 권한을 부여하는 과정 | 특정 리소스에 접근할 수 있는 권한을 부여해 정보를 보호한다.  |
| 쿠키 | 사이트에 방문할 때 사용자에게 저장되는 작은 파일 | 공개되어도 무방한 정보만을 저장하며, 사용자의 브라우저 파일로 저장된다. connectless, stateless상태에서도 개인화 된 서비스를 제공하게끔 해준다.  |
| 세션 | 서버와 클라이언트간 통신에서 상호 인식 후 데이터를 송수신을 마치는 기간 | 공개되면 안되는 정보도 저장하며(예: 로그인), 개인의 데이터를 보호하고 관리해준다. 사용자별 고유식별자(세션아이디)를 생성해 사용자의 정보를 저장한다. 쿠키에 저장된 세션 정보를 통해 연결한다.  |
| 프레임워크,<br>라이브러리 | 프레임워크 : 개발자가 기능을 구현하기 쉽도록 기본적 기능을 제공하는 프로그램의 골조<br>라이브러리 : 소프트웨어 개발시 사용되는 비휘발성 자원들의 집합.프로그램의 추가적인 기능을 구현하는데에 도움이 되는 도구 모음 | 둘다 프로그램을 간편하게 안정적으로 구현하기 위해 사용된다. 다른 점이 있다면 프레임워크는 **지침과 틀**을 통해 가이드라인을 주는 것이고, 라이브러리는 추가를 할 수 있게 수단과 도구를 제공한다는 점이다.  |
| API | 운영체제와 프로그램간의 원활한 통신을 위해 사용되는 언어나 메시지 형식 | 이미 존재하는 애플리케이션 사이에서 통신할 수 있도록 돕는 문서&명세서. 다른 프로그램이 제공하는 기능을 받아와 사용할 때 사용 |
| REST API | REST 아키텍처 스타일을 따르는 응용 프로그램 인터페이스 | 일관된 통신 규칙을 제공하고, 단순한 URL 구조(데이터의 명확한 표현)를 통해 요청의 목적을 쉽게 파악하고 이해하기 쉽도록 한다. 클라이언트-서버를 분리하고, 독립적인 개발과 운영이 가능하도록 한다.  |
| 페이징 | 처리해야할 데이터를 작은 페이지 단위로 분할하여 처리하는 방식 | 서버, 클라이언트의 부하를 줄인다.<br>사용자가 필요한 범위만 보여줄 수 있다.<br>게시판 페이지나 인피니티 스크롤링 등으로 구현이 가능하다.  |
| HTTPS | HTTP 웹 통신 프로토콜을 안전하게 만들어주는 방법 <br>웹을 통해 정보를 안전하게 전송하는 규약 | 최초 접속시 TLS, SSL 같은 인증서(CSR)를 활용해 클라이언트-서버간 상호 인증을 거쳐 연결을 허용한다. 이후 대칭키를 생성해 연결할 수 있다.  |
| 데이터센터 | 서버 컴퓨터나 네트워크 회선등을 제공하는 건물이나 시설 | 온프레미스 데이터 센터와 클라우드 데이터 센터가 존재한다.<br>회사 내에 서버를 두고 사용하면 위험한 점이 많기 때문에, 데이터 센터에서 서버나 회선을 대여하여 사용한다.  |
| log | 통신 소프트웨어 간의 접속 기록이나 메시지, 운영체제나 소프트웨어가 실행되는 중의 이벤트 등의 기록을 작성하여 저장한 파일 | log 확인을 통해 시스템의 에러 확인 및 복구에 사용할 수 있고, 이상현상을 탐지하거나 통계를 내서 분석하는데에 활용할 수도 있다. |

| DOM | HTML 코드가 브라우저에 의해 파싱되어 메모리상에 객체화하여 구성된 구조 | 객체는 html 요소를 나타내고 동작시키는 코드이다. <br>자바스크립트를 사용해 웹페이지의 내용과 구조를 **동적 변경**하기 위해 사용한다.  |
| cssom(css object model) | CSS를 객체화하여 저장한 모델 <br>CSS를 해석해 자바스크립트로 웹페이지의 스타일을 조작하기 위해 사용하는 객체 모델 | 웹 페이지 **스타일을 동적으로** **변경**할 수 있다. |
| event | 사용자가 웹페이지에서 **수행하는 동작** | 사용자와 상호작용하며 동적인 페이지를 만들기 위해 사용한다.  |
| fetch | 비동기적으로 **서버**에게서 데이터를 받아오고 요청하는 과정 | 웹 어플리케이션에서 서버로부터 필요한 데이터를 실시간으로 받을 수 있다.**(새로고침 없이도 동작할 수 있다.)** <br>fetch(url)로 호출하고, .then()으로 응답처리를 할 수있다. .catch()로 오류 처리도 할 수있다.  |
| lottie | json 기반의 애니메이션 파일을 제공해주는 서비스 | 기존 gif 등보다 적은 용량의 벡터 그래픽 애니메이션을 다운 받을 수 있다.  |
| SEO(search engine optimization) | 검색 엔진에서 사이트가 잘 검색될 수(찾아질 수) 있도록 최적화하는 방법 | 사용자들에게 더 많이 노출될 수 있도록 한다. <br>장기적인, 다방면에서의 노력이 필요하다. |
